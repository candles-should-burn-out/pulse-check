# status_set_memberships

Таблица `status_set_memberships`:

| Поле            | Тип           | Constraint | Ограничения                                                                                               |
|-----------------|---------------|------------|-----------------------------------------------------------------------------------------------------------|
| `user_id`       | `TEXT`        | not null   | Первичный ключ; один эффективный набор статусов на аккаунт                                                |
| `status_set_id` | `UUID`        | not null   | Внешний ключ на `status_sets.id` с `ON DELETE CASCADE`; индекс `status_set_memberships_status_set_id_idx` |
| `owner_user_id` | `TEXT`        | not null   | Идентификатор владельца эффективного набора статусов                                                      |
| `created_at`    | `TIMESTAMPTZ` | not null   | Дата создания связи, выставляется backend-ом                                                              |

Дополнительные заметки:

- `user_id` - если пользователь входит без существующей связи, backend создает для него собственный `status_set` и membership владельца.
- `status_set_id` - определяет эффективный набор статусов, который получает пользователь.
- `owner_user_id` - используется для вычисления роли: владелец, если `user_id` равен `owner_user_id`, иначе участник.
