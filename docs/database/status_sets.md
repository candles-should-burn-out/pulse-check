# status_sets

Таблица `status_sets`:

| Поле            | Тип           | Constraint | Ограничения                                                 |
|-----------------|---------------|------------|-------------------------------------------------------------|
| `id`            | `UUID`        | not null   | Первичный ключ, генерируется backend-ом                     |
| `owner_user_id` | `TEXT`        | not null   | Уникальный идентификатор владельца из контекста авторизации |
| `created_at`    | `TIMESTAMPTZ` | not null   | Дата создания записи, выставляется backend-ом               |
| `updated_at`    | `TIMESTAMPTZ` | not null   | Дата последнего изменения записи, обновляется backend-ом    |

Дополнительные заметки:

- `owner_user_id` - один владелец не может иметь больше одного собственного набора статусов.
